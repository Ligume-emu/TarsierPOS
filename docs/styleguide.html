TarsierPOS — Canonical Visual Styleguide

Rebuilt 2026-05-18 · ISSUE-092 · Tokens loaded from frontend/public/shared-styles.css

This page is the visual contract for every shipped UI. The page itself loads tokens from the same shared-styles.css the app uses — if a token changes there, the swatches below change with it. Page chrome (layout, swatch grid, code blocks) lives in this file's <style> and uses only var(--*) tokens. No raw hex, no magic spacing, no Tailwind utilities outside the locked scale.

9 · Dialogs (ISSUE-085)

One confirmDialog(), one alertDialog(). Implemented in frontend/public/dialogs.js. Confirm returns Promise<boolean>; alert returns Promise<void>.

9.1 Confirm — normal

Clear cart?

This removes all items from the current sale. The customer record is not affected.

const ok = await window.confirmDialog({
  title: 'Clear cart?',
  message: 'This removes all items from the current sale.',
  okLabel: 'Clear cart',
});

9.2 Confirm — danger

Void this sale?

Sale #1042 (₱315.00) will be voided. This action is logged and cannot be undone.

const ok = await window.confirmDialog({
  title: 'Void this sale?',
  message: 'Sale #1042 (₱315.00) will be voided.',
  okLabel: 'Void sale',

});

9.3 Alert

Receipt printer offline

The thermal printer at 192.168.1.87 didn't respond. The sale was saved; you can reprint from the X-report screen.

await window.alertDialog({
  title: 'Receipt printer offline',
  message: "The thermal printer didn't respond.",
  icon: '⚠️',
});
Dialog contract (enforced by dialogs.js): backdrop rgba(0,0,0,.5) at --z-modal-backdrop, dialog at --z-modal, focus trap inside the dialog, Esc resolves false (confirm) or closes (alert), backdrop click resolves false, previously-focused element regains focus on close.

22.2 Shift indicator (FEATURE-036 · FEATURE-037)

Replaces the old #shift-bar sticky strip. Renders as a tappable header status pill (#shift-indicator) — a <span> exposed as role="button" with tabindex="0" and aria-haspopup="true", activated by click or Enter/Space. Hydrated by window.renderShiftIndicator(shift) — call this on any currentShift cache change to keep the header in sync.

Active (data-state="open"): shows 🟢 Shift #<id> · <opened_at HH:MM> on a green-tinted background (rgba(34,197,94,0.25)). Inactive (data-state="closed"): shows ⊘ No active shift on the default translucent background (rgba(255,255,255,0.12)). When other accounts have open shifts, the inactive pill appends a floor-state hint · <N> open (from refreshFloorBadge()) without any tap. Tapping the pill (either state) opens the open-shifts floor panel (§22.4) — not the open-shift modal; opening a shift now lives in the dropdown (§22.3). Both states meet the ≥48px touch target via inline min-height: 48px. Source: components/nav.js:202–223 (render), 294–324 (wiring).

22.3 Nav dropdown — open (manager view, shift open)

🛒 POS 📦 Inventory 🧪 Ingredients 📊 Dashboard 📈 X-Report 📉 Z-Report 🔓 Open Shift 🔒 Close Shift 🚪 Logout

Live in components/nav.js:118–123. Settings link is admin-only — omitted for manager. The dropdown is anchored absolute right-0 mt-2 w-48, at --z-dropdown (1020).

Symmetric Open/Close pair — both #nav-open-shift (🔓 Open Shift) and #nav-close-shift (🔒 Close Shift) sit above Logout and are mutually exclusive: renderShiftIndicator() shows Open Shift when no shift is active and Close Shift when one is. Open Shift routes to index.html?open=1 (or invokes window.showOpenShiftModal() directly when already on the POS page); Close Shift routes to zreport.html?close=1. The §22.2 indicator is the one place to read shift state; the dropdown is the one place to change it. Demo above shows the shift-open variant — only one of the pair is visible at a time.

Conditional item pattern — both shift items ship with the hidden class and are toggled by renderShiftIndicator() based on current shift state. This is the canonical "hidden-by-default + state-conditional reveal" pattern for dropdown items: render the element unconditionally, gate visibility via class toggle from the state hydrator. Reuse for upcoming role/page-access-conditional items (e.g. per-account page-access work) rather than rebuilding the dropdown.

22.4 Open shifts floor panel (FEATURE-037)

Cross-account "who's on the floor" view, triggered by tapping the §22.2 shift indicator. Reuses the exact same surface as the ☰ nav dropdown — different content: #shift-panel carries the identical token set absolute right-0 mt-2 w-48 bg-white rounded-lg shadow-xl at --z-dropdown (1020).

👔 jdela_cruz · Shift #41 · opened 07:58 AM
🧑‍🍳 maria_s · Shift #42 · opened 08:15 AM

Each row: <role icon> <username> · Shift #<shift_number> · opened <HH:MM>. Role icons: 👑 admin · 👔 manager · 🧑‍🍳 cashier · 👤 fallback. Empty state renders No open shifts. Visible to all roles; carries no financial fields. Source: GET /api/canteen/shifts/active/ via fetchActiveShifts(), rendered by openShiftPanel() (components/nav.js:264–285).

One overlay at a time — opening the floor panel closes the ☰ dropdown and vice versa; an outside click closes whichever is open. Wired through closeShiftPanel() / toggleShiftPanel() and the shared document click handler (components/nav.js:164–181, 257–292).
